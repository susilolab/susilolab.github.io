Realtime Web?

Realtime web adalah jaringan web yang menggunakan teknologi dan practices yang memungkinkan pengguna memperoleh informasi segera setelah dipublikasikan oleh pembuat.

*wikipedia

Samples

Teknologi

  • Websocket
  • Go
  • Redis
  • Pub/Sub pattern for multi backend

Websocket

protokol komunikasi komputer yang menyediakan saluran komunikasi full-duplex melalui satu koneksi TCP.

Pub/Sub pattern

pola perpesanan dimana pengirim pesan, yang disebut publisher, tidak memprogram pesan yang akan dikirim langsung ke penerima tertentu, yang disebut subscriber, namun mengelompokkan pesan yang dipublikasikan ke dalam kelas tanpa pengetahuan subscriber yg mana.

Konsep

Konsep ini memungkinkan kita membangun aplikasi hybrid, karena dg pola pub/sub dapat digunakan untuk komunikasi antara server/services.

Cara Kerja 1

  • Aplikasi server side akan mengirim pesan ke database (Redis, Mongo) atau GRPC
  • Push system akan meneruskan pesan ke user
  • Web browser meng-update konten atau
  • Web browser meminta data ke "App" lalu meng-update konten

Cara Kerja 2

  • Aplikasi non server side(browser) akan mengirim pesan ke Push System
  • Push System akan meneruskan pesan ke user
  • Web browser meng-update konten atau
  • Web browser meminta data ke "App" lalu meng-update konten

Kelebihan

  • Data dapat dikirim dari server ke klien kapan saja tanpa klien memintanya.
  • Data dapat dikirim dengan cara yang sangat efisien karena hubungan sudah terjalin dan data framenya sangat efisien.
  • Komunikasi 2 arah.
  • Lower latency interaction, artinya delay antara waktu permintaan dan kita mendapatkan respon sangat kecil.

Kekurangan

  • Membutuhkan web browser modern.
  • Lebih komplek dari pada HTTP.
